# 텍스트 엔진 / 폰트 / 렌더러 분석

## 화면 구성 (세이브스테이트 ss1/ss2/ss3 기준)
- 대사는 **BG3** 레이어에 렌더링: 4bpp, char base `0x06008000`, screen base `0x0600F800`, 256×256.
- 배경 장면은 BG0(8bpp). 초상화 등은 OBJ.
- 게임은 **글리프를 런타임에 VRAM 타일로 합성**한다(타일맵으로 배치). 텍스트 박스는 팔레트 인덱스 1(초록) 위에 글자색 인덱스 2/3로 합성.
- 글리프 크기: **8×16** (수직 2타일). 

## 폰트 종류
- **HUD/숫자 폰트: ROM `0x3a3e00`에 무압축 4bpp**로 존재 (숫자 0-9, "HP EN Lv TIME" 등). VRAM과 바이트 일치 확인.
- **메인 대사 폰트: ROM에서 미발견** — 1bpp/4bpp × 여러 크기 전수 스캔, VRAM 글리프의 4bpp/1bpp(글자색 ink) 환원 검색 모두 불일치. → **압축 저장 후 런타임 전개**로 확정적 추정.

## 실행 구조 (동적 분석)
- 코드 일부가 **IWRAM(0x03000000~)으로 복사되어 실행**. 게임 메인 로직 일부는 ROM `0x08025000`, `0x08001000` 등.
- IRQ 핸들러: `0x03000fa0`. 메인 루프는 BIOS VBlankIntrWait(`0x200` 부근 halt)로 프레임 동기화 → 인터럽트 시 PC는 거의 항상 `0x200`(halt).
- 대기/복사 루프: `0x03000d2c`~`0x03000d5c`.

## mGBA GDB 스텁 제약 (중요 — 도구 한계)
- ✅ 메모리 읽기 (단, **한 번에 ≤0x200바이트**), CPU 레지스터, **실행 브레이크포인트(Z0)**, 단일 스텝.
- ❌ **워치포인트(Z2/Z3/Z4) 무효** — `OK` 응답하지만 실제로 안 걸림(IWRAM 쓰기조차). VRAM/스크립트 접근 추적 불가.
- ❌ ROM 읽기 워치 불가.
- 결과: "글리프 합성 순간"을 워치포인트로 못 잡음. 단일스텝 트레이스는 대기 루프에 시간 대부분 소모.

## 동적 분석 추가 결과 (computer-use로 게임 직접 조작)
- **X키(A버튼)로 대사 진행·합성 재현 성공** (free-run 중). ss6에서 "I'm sorry…" → "Ouch! Please let me go!" 합성 확인.
- 정적 대기 시 게임은 **IWRAM 0x03000d28~0x03000d4c 타이트 루프**(VBlank 대기)에 머뭄. 입력 시 여기서 빠져나가 프레임 작업+합성.
- 렌더러 코드는 **IWRAM에서 실행**(ROM 복사본). ROM 0x08000980(1bpp 플로터, dest 0x03001cd4)은 대화 렌더러 아님(합성 시 0히트, IWRAM 미복사).

## 도구 막힘 — 렌더러 PC 포착 불가 (핵심)
세 동적 기법이 모두 막힘:
1. **GDB 워치포인트 무효** — Z2/Z4를 char block(0x06008000)에 걸고 대사 합성(쓰기 발생)을 재현해도 **발화 안 함**. mGBA 네이티브 디버거는 워치포인트 지원하나 **GDB 스텁이 미전달**.
2. **단일스텝 너무 느림** — 12,000스텝 동안 VBlank 대기 루프조차 못 벗어남(프레임당 수만~수십만 명령). 합성 프레임 도달 불가.
3. **mgba-sdl CLI 디버거**(워치포인트 동작) — Windows 콘솔 직접 사용으로 **파이프 구동 불가**(stdin/stdout 리다이렉트 시 OSError).
4. KEYINPUT 메모리 주입 → mGBA가 쓰기 무시(읽기 시 호스트 상태 반환).

→ free-run으로만 프레임 진행 가능 ↔ 렌더러 포착은 중단 필요 → **워치포인트 없이는 IWRAM 렌더러 주소 동적 확정 불가**.

## 렌더러 확정을 위한 권장 경로 (택1)
1. **no$gba** (무료, GBA 디버거 끝판왕) — 메모리 접근 브레이크포인트 동작. char block(0x06008000)에 쓰기 BP → 대사 진행 → 렌더러 PC 즉시 확인. ★가장 빠름
2. **BizHawk** — Lua `event.on_bus_write` 메모리 훅 스크립트화 가능 → char block 쓰기 시 PC 로깅.
3. **Ghidra/IDA 정적 분석** — IWRAM(0x3000000~) + ROM 로드, 0x03001cd4/0x06008000/스크립트 앵커로 렌더러·디컴프레서 역추적. 에뮬 불필요.
4. **수작업 정적 디스어셈블**(capstone) — 0x3000d28 앵커에서 프레임 디스패처 따라 합성 호출 추적. 느리지만 가능.

폰트 소스: raw ROM 무압축 아님 + 표준 LZ77 0건 → 커스텀 압축/런타임 생성. 렌더러를 찾으면 그 입력으로 폰트 디컴프 경로가 드러남.

## 도구
- `scripts/gdbclient.py` — GDB 리모트(읽기 청크 0x200), Z0 BP, 단일스텝.
- `scripts/trace_frame.py` — 프레임 단일스텝 트레이스(r0-r15 캡처).
- VRAM/팔레트/IO 덤프: `out/vram3.bin`, `pal3.bin`, `io3.bin` (ss3 대화 화면).
