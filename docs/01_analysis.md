# ZOE: The Fist of Mars (USA) — 역공학 분석

## 매체 / 식별
- 플랫폼: Game Boy Advance, 카트리지 ROM, 8 MB (8,388,608 bytes)
- 타이틀: `Z.O.E USA`, 게임코드 `AZEE`, 메이커 `A4`(Konami)
- MD5 `b9c0646696297fb21cfad315c434cc2a`, CRC32 `24244415`
- 원작: ZONE OF THE ENDERS 2173 TESTAMENT (JP). USA판은 **영어**이므로 EN→KO 번역.

## 텍스트 인코딩 (확정)
무압축. 1바이트 커스텀 폰트 테이블:

| 바이트 | 문자 |
|--------|------|
| 0x20 | space |
| 0x21–0x3a | `A`–`Z` |
| 0x3b–0x54 | `a`–`z` |

### 제어 코드
- `0x01 NN` — 문장부호/제어 (확정): `12`=`.` · `13`=`,` · `17`=`'` · `0c`=`!` · `0d`=`?` · `28`=`-` · `19`=`{nl}`(개행) · `1d`=`{name}`(이름/대사 구분) · 그 외 `15,1c,22,0f…`는 미확정(보존 토큰)
- `0x15 NN` — 숫자/변수 삽입 (`{var:NN}`)
- `0x1f` — 레코드(대사 1턴) 종료
- 레코드 구조: `<이름><1d><19><대사 본문><1f>` 또는 본문만

## 스크립트 위치 (정정)
- **스토리 대사: 0x90000 – 0x17a000** (64KB 블록당 ~550 레코드 균일)
- **이름/아이템/캐릭터 바이오: 0x70000 – 0x90000** (타이틀 텍스트 0x78400)
- **0x3a000 – 0x70000 은 그래픽 데이터**(텍스트 아님 — 초기 추정 오류 정정)
- 추출 결과(0x70000+): **17,811 레코드 / 763,896자** → `assets/translations/raw/script.json`
  - 그중 공통 영단어 포함 "확실 대사" 7,561 레코드 / 684K자
- **코덱 라운드트립 무손실 검증 완료**: 18k 레코드 전부 decode→encode→원본 일치 (`scripts/codec_roundtrip.py`)

## 포인터
- 절대 32비트(`0x08xxxxxx`) 스캔은 그래픽 오탐만 다수 → 스크립트는 **블록 로컬 상대 오프셋** 방식으로 추정 (예: 0x78462 테이블 = 블록 기준 상대값 0x400,0x3fd…). 재삽입 전 정밀 확정 필요.

## 폰트 (미해결 — 핵심 블로커)
- raw ROM에서 1bpp/4bpp × 8x8/8x16/16x16 전수 스캔 → **폰트 미발견**.
- 결론: 폰트가 **압축 저장 후 런타임에 VRAM으로 전개**되는 것으로 추정 (가이드 PDF "압축 데이터" 정황과 일치).
- 확인 경로: mGBA(0.10.5) + GDB 스텁으로 VRAM(0x06000000) 덤프 파이프라인 구축 완료.
  단, 게임이 **입력 대기**라 인트로/타이틀에서 텍스트 화면에 자동 도달하지 못함 → 에뮬레이터에 입력을 넣어 대사 화면에서 VRAM 캡처해야 폰트 포맷·렌더 경로 확정 가능.

## 도구 / 환경
- Python 3.12 + PIL. mGBA 0.10.5 (`C:\Program Files\mGBA`). CrystalTile2, 폰트테이블생성기 (tools/).
- 스크립트: `scripts/zoektext.py`(코덱), `extract_text.py`(추출), `tiledump.py`(타일렌더), `gdbclient.py`(VRAM 덤프).
